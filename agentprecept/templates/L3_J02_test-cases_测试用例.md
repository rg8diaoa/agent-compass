## §1 用例格式

```
给定 {前置条件}
当   {触发行为}
则   {期望结果}
```

## AGENT: 填充流程

```
1. 读取已有测试文件 → 提取所有现有用例（解析给定/当/则段落）
2. 读取 L2_G01 API 端点列表 → 对比 → 找出未覆盖的端点
3. 为每个未覆盖端点生成 2 个用例骨架: 正常 + 异常
4. 读取 L2_F04（异常处理）异常类型 → 为每种异常生成边界用例
5. 输出报告: "已覆盖 X 个端点 / 未覆盖 Y 个端点 / 建议新增 Z 个用例"
```

## 完成标准

□ 每端点 ≥ 2 用例（正常+异常）
□ 边界条件清单非空
□ 未覆盖端点 = 0

## 反模式

| 不要 | 要 |
|------|------|
| 只写 happy path | 正常 + 异常 + 边界 三条线 |
| 用例名叫 `test1` | `TC-AUTH-01：正常登录` |
| API 变了用例不同步 | PR 须附用例变更清单 |

## Agent 规则

1. 新功能 → 先列用例再写代码（TDD）
2. 用例命名：`TC-{MODULE}-{NN}：{简述}`
3. API 变了 → 用例同步更新
4. 每个 PR 附带新增/修改的用例清单
