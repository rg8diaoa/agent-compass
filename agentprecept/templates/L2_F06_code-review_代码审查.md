## Agent 代码审查检查清单

Agent 在提交 PR 或审查他人代码时，按此清单逐项检查。

### 正确性

- □ 逻辑是否正确——不只是"看起来没问题"
- □ 边界条件是否覆盖（空输入、null、超长字符串）
- □ 错误路径是否被正确处理（不只是 happy path）
- □ 是否引入了竞态条件（多线程/异步场景）

### 可维护性

- □ 命名是否符合 L1_A02 规范
- □ 函数是否超过 50 行（超过 → 建议拆分）
- □ 是否引入了新的设计决策（→ 追加 L4_O01）
- □ 是否修改了已有模块的公开接口（→ 检查 project-graph 影响范围）

### 安全性

- □ 是否有硬编码密钥/密码/token
- □ 数据库查询是否参数化
- □ 新 API 端点是否有认证检查
- □ 用户输入是否有验证

### 测试

- □ 新模块是否有对应测试文件
- □ 修改是否影响了已有测试（→ 先跑全量测试）
- □ 关键路径是否有测试覆盖

### 文档同步

- □ 新模块是否更新了 project-graph.yaml
- □ API 变更是否更新了 L2_G01
- □ 配置变更是否更新了 L2_E03 和 .env.example

---

## Agent 审查流程

```
1. 读 project-graph.yaml → 理解改动的影响范围
2. 逐项过检查清单
3. 发现问题 → 标注严重程度（🔴阻塞 🟡建议）
4. 输出审查意见 → 每条问题附修复建议
```
