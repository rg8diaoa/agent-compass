# M1_A02 — 自然语言编程指南

> 状态: 📝撰写中 | 目标读者: 非技术用户

> 你不会写代码，但你想用 AI Agent 帮你做应用。这一篇教你怎么用自然语言说清楚你想要什么——Agent 理解错，往往是你没说清楚。

---

## 需求三要素

每次描述需求，至少说清三件事：

| 要素 | 问自己 | 例子 |
|------|--------|------|
| 谁用 | 这个应用给谁用？几个人？ | "我一个人用的日记" vs "小团队 5 人的任务看板" |
| 做什么 | 核心功能是什么？只列最重要的 1-3 个 | "记录文字 + 按日期浏览 + 搜索" |
| 存哪里 | 数据需要保存吗？跨设备吗？ | "只在我电脑上" vs "手机和电脑都能看" |

三要素缺一不可。Agent 能从这三个答案推断技术方案。

## 用日常语言描述

**不要**：
> "我需要一个 RESTful API，后端用 FastAPI，前端 React，数据库 PostgreSQL"

**要说**：
> "我想做一个日记应用，每天写一段话，能按日期翻看，能全文搜索。就我一个人用，装在我电脑上就行。"

Agent 会自己选技术栈。你是业主，不是包工头。

## 常见坑

| 你的说法 | Agent 的困惑 | 更好的说法 |
|----------|-------------|-----------|
| "做个商城" | 卖什么？谁卖？支付怎么处理？ | "二手书交易网站：发布书→买家留言→线下交易。不做在线支付。" |
| "要能登录" | 邮箱？手机？微信？几个人？ | "用邮箱注册登录，找回密码用邮箱验证码" |
| "界面好看点" | 什么算好看？有参考吗？ | "像 Notion 那种简洁风，白底黑字，不要太花" |

## 第一轮只做最小版本

```
你: "做个博客"
Agent: 拆出 15 项 checklist

❌: 全部确认，让 Agent 全做
✅: "先做第 1-3 项：能写文章、显示文章列表、点开看全文。
      其他的看完效果再说"
```

原因：第一版没跑通之前，你不知道哪些需求其实不需要、哪些想法会变。先看到能跑的东西，再迭代。

## 信任校准

| 阶段 | 怎么看 | 信多少 |
|------|--------|:--:|
| Agent 复述你的需求 | 看它理解的对不对 | 可以信——它只是换了个说法 |
| Agent 出 checklist | 看有没有漏功能 | 多半对——检查遗漏 |
| Agent 出设计文档 | 看模块名是否合理 | 存疑——你可能不懂，等跑起来再看 |
| Agent 写完代码 | **跑起来试** | 不盲信——功能都点一遍 |
| Agent 说"完成了" | **对照需求逐条试** | 验证完再信 |

**核心原则**：能跑起来的功能 > Agent 说"写好了"。

## 与 AgentPrecept 的衔接

Agent 加载 AGENTS.md 后会自动：
1. 拆你的需求为 checklist
2. 出架构设计草稿 → 标注 [NEEDS_HUMAN_REVIEW] → 等你确认
3. 确认后开始写代码
4. 每完成一项更新 HANDOFF

你只需要做三件事：说清楚需求 → 确认设计草稿 → 跑起来验证。
