## 设计决策

| 决策 | 来源 | 证据 |
|------|:--:|------|
| 为什么用三层架构（api/services/models）而不是两层 | 2025-01 架构评审 | 两层导致 api 层混入业务逻辑，service 层可独立测试 |
| 为什么用 JWT 而不是 API Key | 2025-01 安全评估 | JWT 无状态，分布式部署无需共享密钥；API Key 需要额外撤销机制 |
| 为什么用 PostgreSQL 而不是 MongoDB | 2025-01 数据建模 | task-user 的关系查询在关系型中更自然，事务支持更好 |
| 为什么测试覆盖率目标 80% 而不是 90% | B4 项目教训 | B4 在 38% 覆盖率时漏 18 个线上 bug；80% 足以覆盖核心路径，90% 的边际收益骤降 |
